To ensure that your system logs the original entry data before any major user actions like edits or deletions for auditing purposes, you can implement an audit trail feature. This will involve creating an audit_logs table in your database that captures relevant details whenever modifications occur within your major modules (e.g., medical visits and incident reporting).

Here’s how you can implement this:

    Create an Audit Log Table: This table will store details about each action—what was done, who did it, when it was done, and any relevant details.

    This is similar to what you already have in your

export const auditLogs = pgTable("audit_logs", {
  id: varchar("id").primaryKey().default(sql`gen_random_uuid()`),
  tenantId: varchar("tenant_id").notNull().references(() => tenants.id, { onDelete: "cascade" }),
  userId: varchar("user_id").notNull().references(() => users.id, { onDelete: "cascade" }),
  action: varchar("action").notNull(), // create, update, delete, etc.
  resourceType: varchar("resource_type").notNull(), // e.g., medical_visit, incident
  resourceId: varchar("resource_id"), // ID of the affected resource
  originalData: jsonb("original_data"), // Original data before update or delete
  details: jsonb("details"), // Additional context
  createdAt: timestamp("created_at").defaultNow(),
});

Capture Changes: Modify the existing handlers for updates and deletes to log the original data into this table right before making the changes.

For example, in your handlers for medical visit and incident updates, you would add a section to insert into the audit_logs table:

// Update medical visit
app.put('/api/medical-visits/:id', hybridAuthMiddleware, async (req, res) => {
  const visitId = req.params.id;
  const newData = req.body;

  // Fetch original data
  const originalData = await storage.getMedicalVisit(visitId);

  // Log original data before updating
  await storage.insertAuditLog({
    userId: req.user.id,
    action: 'update',
    resourceType: 'medical_visit',
    resourceId: visitId,
    originalData: originalData,
    details: newData, // You can include details of the change here
  });

  // Proceed with actual update
  const updatedVisit = await storage.updateMedicalVisit(visitId, newData);
  res.json(updatedVisit);
});

By implementing this audit trail feature, you ensure that original data is preserved before any important changes, allowing for a robust auditing mechanism across your application's major functionalities.
